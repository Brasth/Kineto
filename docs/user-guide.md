# Kineto User Guide

## Overview

Kineto is a local-first macOS meeting assistant. It captures a user-selected application or display, optionally captures the microphone as a separate `You` track, transcribes locally with Whisper, translates finalized English and Vietnamese segments, and creates an evidence-linked summary after recording stops.

The current build is a development build for Apple Silicon Macs running macOS 26.1 or newer. It is not a signed or notarized public installer.

## Prerequisites

- Apple Silicon Mac
- macOS 26.1 or newer
- Full Xcode 26.6 selected with `xcode-select`
- Screen & System Audio Recording permission
- Microphone permission when the microphone option is enabled
- The verified Whisper model included in this repository

## Open Kineto

### Recommended: run from Xcode

From Terminal:

```bash
cd /Volumes/1TB/Projects/Kineto
open Kineto.xcodeproj
```

In Xcode:

1. Select the **Kineto** scheme.
2. Select **My Mac** as the destination.
3. Press **Command-R**.

### Open an existing local build

Debug build:

```bash
open -n ~/Library/Developer/Xcode/DerivedData/Kineto-*/Build/Products/Debug/Kineto.app
```

Release build:

```bash
open -n ~/Library/Developer/Xcode/DerivedData/Kineto-*/Build/Products/Release/Kineto.app
```

The random directory suffix under `DerivedData` is generated by Xcode and can change after cleaning build data.

## Import the Whisper model

Kineto accepts one exact pinned model:

```text
/Volumes/1TB/Projects/Kineto/Models/ggml-large-v3-turbo-q5_0.bin
```

To import it:

1. Open Kineto and select **New Meeting**.
2. Select **Import verified model…**.
3. In the file picker, press **Command-Shift-G**.
4. Enter `/Volumes/1TB/Projects/Kineto/Models`.
5. Select `ggml-large-v3-turbo-q5_0.bin`.

Do not select the `.sha256` file, `CWhisper.xcframework`, or the entire `Models` directory. Kineto copies the selected `.bin` file into its sandbox only after checking its exact byte count and SHA-256 digest.

A successful import first displays **Loading Whisper model into memory…**, then **Whisper model ready · local**. Meeting start stays disabled until the one-time local model warm-up completes.

## Grant macOS permissions

Open **System Settings → Privacy & Security** and review:

- **Screen & System Audio Recording**: required for selected-application or display audio.
- **Microphone**: required only when **Include my microphone as “You”** is enabled.

If macOS asks for an app restart after granting permission, quit Kineto completely and launch it again. Never bypass the system source picker; the picker is the authoritative capture boundary.

## Start a meeting

1. Open the meeting application or browser window that will supply audio.
2. Select **New Meeting**.
3. Select **Choose…** under **Capture source**.
4. In Apple's sharing picker, choose the meeting application or an entire display and confirm sharing. Kineto excludes itself so it cannot accidentally capture a silent self-source.
5. Keep **Include my microphone as “You”** enabled only if the microphone track is needed.
6. Enable English ↔ Vietnamese translation if desired.
7. Wait for **Translation assets ready**. Kineto prepares both language directions before enabling a translation-enabled start.
8. Enable the post-meeting summary if desired and choose English or Vietnamese.
9. Confirm that participants were informed and acknowledge the displayed capture boundary.
10. Select **Start Meeting**.

Application capture may include every audible window from that application. Display capture may include all audio associated with that display. Kineto does not claim browser-tab isolation or meeting-platform participant identity.

## During recording

- **Selected Source** represents captured application or display audio.
- **You** represents the optional microphone.
- Finalized English and Vietnamese segments are translated locally when translation is enabled.
- A visible transcript gap means audio could not be safely processed; Kineto does not fabricate text across missing audio.
- Use **Pause** and **Resume** to suspend and continue capture.
- Use **Stop** to finish capture, drain finalized transcription, seal the source meeting, and generate the optional summary.

## Review, export, and delete

Completed meetings appear in the meeting library.

- Select a meeting to reopen its transcript and summary. Use **Summary** for the generated evidence-linked review or **Ask** for a dedicated grounded conversation view; chat is unavailable during capture and processing. The question field supports up to 1,500 characters: **Return** adds a line; **Command-Return** or **Send** submits it. Questions and saved answers stay on this Mac in the encrypted meeting package.
- Grounded chat answers link exact source excerpts. If Kineto cannot find relevant final transcript support, or the local model is unavailable, it displays an explicit no-answer rather than guessing.
- Select an **Evidence** link to inspect the exact source segment supporting a summary item or chat answer.
- **Export Plaintext…** creates a user-selected JSON copy, including saved chat turns. The exported copy is outside Kineto's encryption and deletion boundary.
- **Delete…** removes Kineto's meeting keys before best-effort removal of encrypted package bytes and encrypted chat history. Plaintext exports must be deleted separately.

Raw audio retention is off in the current application workflow.

## Troubleshooting

### An older build crashes after recording begins

Quit every running Kineto instance, rebuild, and open the newly built application:

```bash
cd /Volumes/1TB/Projects/Kineto
xcodebuild \
  -project Kineto.xcodeproj \
  -scheme Kineto \
  -configuration Debug \
  -destination 'platform=macOS,arch=arm64' \
  CODE_SIGNING_ALLOWED=NO \
  build
open -n ~/Library/Developer/Xcode/DerivedData/Kineto-*/Build/Products/Debug/Kineto.app
```

The July 18 recording crash was caused by retaining a SwiftUI `translationTask` session after its task closure completed. Current code uses that session only for in-scope asset preparation and creates actor-owned installed-language sessions for live translation.

### Start Meeting remains disabled

Verify all of the following:

- A capture source is selected.
- The Whisper model reports **ready · local**.
- Participant acknowledgment is selected.
- Translation reports **assets ready**, or translation is disabled.
- No model import or other setup operation is still running.

### The picker does not show the meeting application

Open the target application and at least one visible meeting window before selecting **Choose…**. Apple's picker lists currently shareable applications, windows, and displays. Kineto itself is intentionally excluded.

### Repeated “Thank you” text appears during silence

That output was a Whisper hallucination from silent recognition chunks, especially when Kineto itself had been selected while its own process audio was excluded. Current builds reject silent/brief-noise chunks, isolate decoder context between independent chunks, and prevent Kineto from appearing as a capture source.

### No microphone transcript appears

- Confirm **Include my microphone as “You”** was enabled before start.
- Confirm Kineto has Microphone permission.
- Check the selected input device in macOS Sound settings.
- Restart Kineto after changing permission.

### No selected-source audio appears

- Confirm Kineto has Screen & System Audio Recording permission.
- Reopen the system picker and select the intended application or display.
- Confirm that the selected source is producing audible output.
- Remember that Kineto excludes its own process audio.

### Translation is unavailable

Connect to the network only if macOS needs to install Apple's language assets, reopen preflight, and wait for both directions to report ready. Once installed, the intended meeting workflow remains local. Translation currently supports English ↔ Vietnamese only.

## Current limitations

- English ↔ Vietnamese translation only
- Chinese may be recognized by Whisper but Chinese translation and summaries are not supported
- No cloud AI providers, accounts, OAuth, or API-key workflow
- No browser-tab isolation or meeting-platform bot
- No retained raw-audio package
- No signed/notarized public DMG yet

Release blockers and external validation requirements are tracked in `docs/project-roadmap.md` and `docs/deployment-guide.md`.
